# A Multi-Stage Machine Learning Framework for Network Intrusion Detection

GitHub repository: https://github.com/CSweaker/441Project

## Project Summary

This project develops an anomaly-based Intrusion Detection System (IDS) using the NSL-KDD benchmark dataset. Traditional signature-based IDS tools can miss zero-day or evolving attacks because they depend on known attack patterns. This project instead uses supervised machine learning to distinguish benign traffic from malicious traffic and then classify malicious traffic into major attack families.

The final system is designed as a two-stage IDS pipeline. Stage 1 performs binary classification: normal traffic vs. attack traffic. Stage 2 is only applied to records predicted as attacks and classifies them into DoS, Probe, R2L, and U2R. The project compares baseline models, including Dummy Classifier, Gaussian Naive Bayes, and Logistic Regression with PCA, against a stronger tree-based Random Forest model. A Streamlit app is included as the demo interface.

## Methodology

The workflow includes:

1. **Data loading** from NSL-KDD `KDDTrain+.txt` and `KDDTest+.txt`.
2. **Preprocessing** with log transformation for skewed byte/count variables, standard scaling for numerical features, and one-hot encoding for categorical features (`protocol_type`, `service`, and `flag`).
3. **Statistical analysis** required by the project rubric:
   - Central Limit Theorem demonstration using repeated sampling of `src_bytes`.
   - Mann-Whitney U test and Welch t-test comparing normal vs. attack traffic.
   - Bootstrap 95% confidence interval for the difference in mean log-transformed `src_bytes`.
4. **Baseline modeling**:
   - Dummy majority classifier.
   - Gaussian Naive Bayes with PCA.
   - Logistic Regression with PCA.
5. **Improved model**:
   - Random Forest for Stage 1.
   - Extra Trees classifier for Stage 2 attack family classification.
6. **Evaluation**:
   - Accuracy, macro F1, weighted F1, classification reports, and confusion matrices.
7. **Deployment demo**:
   - Streamlit app for uploading NSL-KDD-style files and viewing predictions.

## How to Run

Recommended Python version: Python 3.10 or newer.

Install dependencies:

```bash
pip install -r requirements.txt
```

Run a faster smoke test first:

```bash
python train_pipeline.py --quick --sample-size 10000 --test-sample-size 3000
```

Run the automated sanity check:

```bash
python run_sanity_check.py
```

Train the full project:

```bash
python train_pipeline.py --train data/KDDTrain+.txt --test data/KDDTest+.txt
```

Launch the demo app after training:

```bash
streamlit run app.py
```

## Important Artifact Note

The package includes `artifacts/ids_artifacts.joblib` so the Streamlit demo can be tested quickly. Treat this as a demo/quick-run artifact. For the final reproducible experiment, rerun full training with:

```bash
python train_pipeline.py --train data/KDDTrain+.txt --test data/KDDTest+.txt
```

Only load `.joblib` artifacts generated by this project or another trusted source.

## Optimizations in This Version

Compared with the earlier package, this version:

- Adds the required GitHub repository link in the README.
- Adds `seaborn>=0.13` because the legacy baseline script imports it.
- Uses script-relative paths in `app.py`, `train_pipeline.py`, and `run_sanity_check.py`, so commands are less dependent on the current working directory.
- Reads Streamlit uploads from memory instead of writing leftover temporary files.
- Restricts zip extraction to official NSL-KDD files only: `KDDTrain+.txt` and `KDDTest+.txt`.
- Makes stratified quick sampling exact-size, avoiding small off-by-one row count differences.
